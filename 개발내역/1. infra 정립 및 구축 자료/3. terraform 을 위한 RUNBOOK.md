# analysis-agent-infra Terraform 런북

이 문서는 Analysis Agent 인프라를 처음 배포하거나
팀원이 이어받아서 작업할 때 참고하는 운영 문서입니다.

---

## 사전 조건

| 항목 | 확인 방법 |
|---|---|
| Terraform 설치 (>= 1.5.0) | `terraform --version` |
| AWS CLI 설치 | `aws --version` |
| AWS 자격증명 설정 | `aws sts get-caller-identity` |
| AWS 계정 ID 확인 | `aws sts get-caller-identity --query Account --output text` |

## 최초 배포 순서

### Step 1. 레포 클론 및 폴더 이동

```bash
git clone https://github.com/ACS-Nutrients/codecaine-terraform-infra.git
cd codecaine-terraform-infra/analysis-agent-infra
```

### Step 2. handler.py 복사

Lambda 소스 파일을 `files/` 폴더에 복사합니다.
이 파일은 `.gitignore`에 등록되어 있어 Git에 올라가지 않으므로
매번 수동으로 복사해야 합니다.

```bash
cp ../../analysis-agent-ver1/lambdas/action_nutrient_calc/handler.py ./files/handler.py
```

> ⚠️ 이 파일이 없으면 `terraform apply` 시 아래 에러 발생
> `Error: source_file ... no such file or directory`

### Step 3. terraform.tfvars 확인

```bash
cat terraform.tfvars
```

아래 값들이 올바른지 확인합니다.

```hcl
aws_region     = "ap-northeast-2"
aws_account_id = "070238434919"
project_name   = "analysis-agent"
environment    = "dev"
github_org     = "ACS-Nutrients"
github_repo    = "analysis-agent-ver1"
```

### Step 4. terraform init

```bash
terraform init
```

정상 출력:
```
Terraform has been successfully initialized!
```

### Step 5. terraform plan

```bash
terraform plan
```

생성될 리소스 목록 확인:

| 리소스 | 이름 |
|---|---|
| `aws_ecr_repository` | analysis-agent |
| `aws_lambda_function` | analysis-agent-nutrient-calc |
| `aws_iam_role` | analysis-agent-lambda-role |
| `aws_iam_role` | analysis-agent-agentcore-runtime-role |
| `aws_iam_role` | analysis-agent-github-actions-role |
| `aws_iam_openid_connect_provider` | GitHub OIDC |
| `aws_bedrockagentcore_runtime` | analysis_agent |

> ⚠️ CLI로 이미 만든 리소스가 있으면 충돌 발생 가능
> 아래 "기존 리소스 충돌 처리" 섹션 참고

### Step 6. terraform apply

```bash
terraform apply
```

`yes` 입력 후 완료까지 대기 (약 2~5분 소요).

### Step 7. output 값 확인 및 설정

```bash
terraform output
```

출력값을 각각 아래 위치에 설정합니다.

| output | 설정 위치 |
|---|---|
| `agentcore_runtime_arn` | App 서비스 환경변수 `AGENTCORE_RUNTIME_ARN` |
| `github_actions_role_arn` | GitHub Secrets `AWS_DEPLOY_ROLE_ARN` |
| `ecr_repository_url` | 참고용 (GitHub Actions에서 자동 사용) |

## 기존 리소스 충돌 처리

CLI로 이미 만든 리소스가 있을 때 `terraform plan`에서 충돌이 발생하면
`terraform import`로 기존 리소스를 state에 가져옵니다.

```bash
# ECR
terraform import aws_ecr_repository.analysis_agent analysis-agent

# Lambda
terraform import aws_lambda_function.nutrient_calc action-nutrient-calc

# IAM Role - lambda
terraform import aws_iam_role.lambda_exec lambda-basic-role

# IAM Role - agentcore
terraform import aws_iam_role.agentcore_runtime agentcore-runtime-role

# IAM Role - github actions
terraform import aws_iam_role.github_actions github-actions-analysis-agent

# AgentCore Runtime
terraform import aws_bedrockagentcore_runtime.analysis_agent analysis_agent-bA2YOVDhUj
```

import 후 다시 `terraform plan`으로 drift 확인 후 `apply`.

## 업데이트 배포 순서

Lambda 코드가 변경된 경우:

```bash
# 1. handler.py 다시 복사
cp ../../analysis-agent-ver1/lambdas/action_nutrient_calc/handler.py ./files/handler.py

# 2. plan & apply
terraform plan
terraform apply
```

AgentCore Runtime 이미지 업데이트는 GitHub Actions(deploy.yml)가 담당하므로
Terraform에서 별도 작업 불필요.

## 트러블슈팅

| 상황 | 원인 | 해결 |
|---|---|---|
| `source_file no such file or directory` | files/handler.py 없음 | Step 2 handler.py 복사 실행 |
| `EntityAlreadyExists` | 동일 이름 IAM Role 이미 존재 | terraform import 후 재시도 |
| `ResourceConflict` ECR | ECR 레포 이미 존재 | terraform import 후 재시도 |
| `ValidationException` AgentCore | ECR 이미지 없는 상태에서 Runtime 생성 | ECR에 이미지 먼저 푸시 후 재시도 |
| `plan`에서 agentcore 이미지 변경 감지 | lifecycle ignore_changes 미작동 | agentcore.tf lifecycle 블록 확인 |
