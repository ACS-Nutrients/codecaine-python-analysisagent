# GitHub Actions CI/CD 가이드

---

## 개요

deploy 브랜치에 merge되면 자동으로 빌드 → ECR 푸시 → AgentCore Runtime 업데이트가 실행됩니다.

> Lambda는 GitHub Actions에서 배포하지 않음.
> Terraform이 인프라 배포 시점에 함께 배포하는 것이 맞음.
> 이유: Lambda는 인프라성 리소스로 코드 변경이 거의 없고,
> 이미지 빌드도 필요 없기 때문에 별개의 배포 전략을 사용.

---

## 파이프라인 흐름

```
deploy 브랜치 push (PR merge)
  │
  ├─ [1] test              pytest tests/ -v
  ├─ [2] build-and-push    arm64 Docker 빌드 → ECR 푸시
  └─ [3] deploy-agentcore  AgentCore Runtime 이미지 업데이트 → READY 대기
```

2번과 3번은 순차 실행 (3번은 2번 완료 후 실행).


## 인증 방식 — OIDC

AWS Access Key를 GitHub Secrets에 저장하지 않고
OIDC 토큰으로 AWS IAM Role을 임시 assume하는 방식.

```
GitHub Actions 실행
  └─► OIDC 토큰 발급 (github.com)
        └─► AWS STS AssumeRoleWithWebIdentity
              └─► 임시 자격증명 발급
                    └─► ECR 푸시, AgentCore 업데이트 실행
```

장점: AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY를 GitHub에 저장할 필요 없음.

## 최초 1회 설정 순서

### 1. CI/CD 파일을 main 브랜치에 먼저 추가

```bash
mkdir -p .github/workflows
mkdir -p scripts
# deploy.yml, setup_oidc.py 복사 후

git add .github/workflows/deploy.yml scripts/setup_oidc.py
git commit -m "chore: add CI/CD pipeline"
git push origin main
```

> ⚠️ deploy 브랜치가 없는 상태에서 파일을 추가하므로
> main에 먼저 올려서 파일을 준비한 뒤 OIDC 설정 진행.

### 2. OIDC 설정 스크립트 실행

```bash
python3 scripts/setup_oidc.py \
  --github-org  ACS-Nutrients \
  --github-repo analysis-agent-ver1
```

스크립트가 자동으로 생성하는 것들:
- GitHub OIDC Provider (AWS IAM에 등록)
- `github-actions-analysis-agent` IAM Role
- 배포에 필요한 최소 권한 정책

### 3. GitHub Secrets 등록

```
레포 → Settings → Secrets and variables → Actions
→ New repository secret (Repository secret으로 등록)

Name:   AWS_DEPLOY_ROLE_ARN
Secret: arn:aws:iam::070238434919:role/github-actions-analysis-agent
```

### 4. deploy 브랜치 생성 & push → Actions 자동 실행

```bash
git checkout -b deploy
git push origin deploy
# → 이 순간 GitHub Actions 자동 트리거
```

## GitHub Actions에 필요한 IAM 권한

`github-actions-analysis-agent` Role에 부여된 권한:

| 권한 | 용도 |
|---|---|
| `ecr:GetAuthorizationToken` | ECR 로그인 |
| `ecr:BatchCheckLayerAvailability` | 이미지 레이어 확인 |
| `ecr:PutImage` | 이미지 푸시 |
| `ecr:InitiateLayerUpload` | 레이어 업로드 시작 |
| `ecr:UploadLayerPart` | 레이어 업로드 |
| `ecr:CompleteLayerUpload` | 레이어 업로드 완료 |
| `bedrock-agentcore-control:UpdateAgentRuntime` | AgentCore 이미지 업데이트 |
| `bedrock-agentcore-control:GetAgentRuntime` | AgentCore 상태 확인 |
| `iam:PassRole` | agentcore-runtime-role 전달 |

## 주의사항

| 항목 | 내용 |
|---|---|
| `python` 명령어 | Mac은 `python3` 사용 |
| Secrets 종류 | Environment secret 아닌 **Repository secret** 으로 등록 |
| deploy 브랜치 생성 시점 | OIDC + Secrets 등록 완료 후 생성해야 Actions 성공 |
| Docker 아키텍처 | AgentCore Runtime은 **arm64** 필수 |
| Lambda 배포 | GitHub Actions에서 하지 않음 → Terraform 담당 |
