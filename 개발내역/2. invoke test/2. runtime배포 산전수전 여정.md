
## 문제 4. ECR 권한 없어서 AgentCore 배포 실패

**에러**
```
Access denied while validating ECR URI
```

**원인**
agentcore-runtime-role에 ECR 이미지 풀 권한 없음.

**해결**
```bash
aws iam put-role-policy \
  --role-name agentcore-runtime-role \
  --policy-name ecr-pull \
  --policy-document '{
    "Statement": [{
      "Effect": "Allow",
      "Action": ["ecr:GetAuthorizationToken","ecr:BatchGetImage","ecr:GetDownloadUrlForLayer"],
      "Resource": "*"
    }]
  }'
```

## 문제 6. chromadb 빌드 실패 (C++ 컴파일러 없음)

**에러**
```
RuntimeError: Unsupported compiler -- at least C++11 support is needed!
ERROR: Failed building wheel for chroma-hnswlib
```

**원인**
python:3.12-slim 이미지에 C++ 컴파일러 없음.

**해결**
```dockerfile

    && rm -rf /var/lib/apt/lists/*
```

## 문제 8. chromadb DB 버전 불일치 (collections.topic 에러)

**에러**
```
sqlite3.OperationalError: no such column: collections.topic
```

**원인**

**해결**
```

```

## 문제 9. chromadb config_json_str 빈 값 ← 핵심 난관

**에러**
```
KeyError: '_type'
TypeError: object of type 'int' has no len()
ValueError: Invalid parameter name: max_elements
```

**원인**
크롤링 스크립트가 ChromaDB collection 생성 시
`config_json_str`을 `{}` 빈 값으로 저장.
chromadb 0.5.0이 DB 로드 시 `_type` 키를 찾지 못해 에러.

**해결 과정**
```bash
# 1. DB 스키마 확인
sqlite3 lpi_vector_db/chroma.sqlite3 ".schema collections"

# 2. config_json_str 확인
sqlite3 lpi_vector_db/chroma.sqlite3 \
  "SELECT name, config_json_str FROM collections;"
# 결과: lpi_interactions|{}  ← 빈 값이 문제

# 3. 직접 수정 (max_elements 제외 - 이 버전 미지원)
sqlite3 lpi_vector_db/chroma.sqlite3 \
  "UPDATE collections SET config_json_str = '{
    \"_type\": \"CollectionConfigurationInternal\",
    \"hnsw_configuration\": {
      \"_type\": \"HNSWConfigurationInternal\",
      \"space\": \"cosine\",
      \"ef_construction\": 100,
      \"ef_search\": 100,
      \"num_threads\": 4,
      \"resize_factor\": 1.2,
      \"batch_size\": 100,
      \"sync_threshold\": 1000,
      \"M\": 16
    }
  }' WHERE name = 'lpi_interactions';"
```

> ⚠️ `max_elements` 파라미터는 이 버전에서 지원 안 함 → 넣으면 또 에러

## 문제 10. 멀티스테이지 빌드 경로 문제

**원인**
```dockerfile
# 이렇게 하면 /root/.local 경로 문제 발생
COPY --from=builder /root/.local /root/.local
```

**해결**
단일 스테이지 빌드로 변경.
```dockerfile

RUN apt-get update && apt-get install -y gcc g++ && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
```

## 문제 11. AgentCore Runtime 환경변수 미설정

**에러**
```
RuntimeClientError: Received error (500) from runtime.
```

**원인**
`update-agent-runtime` 시 `--environment-variables` 누락.
컨테이너에 OPENAI_API_KEY 등이 없어서 500 발생.

**해결**
```bash
aws bedrock-agentcore-control update-agent-runtime \
  --agent-runtime-id analysis_agent-bA2YOVDhUj \
  ... \
  --environment-variables '{"LLM_PROVIDER":"openai","OPENAI_API_KEY":"sk-...","OPENAI_MODEL_ID":"gpt-4o","AWS_REGION":"ap-northeast-2","LAMBDA_FUNCTION_NAME":"action-nutrient-calc","KB_LOCAL_PATH":"/app/lpi_vector_db","KB_COLLECTION_NAME":"lpi_interactions","KB_TOP_K":"3"}'
```

> ⚠️ 환경변수는 Runtime에 영구 저장됨.
> update 시 매번 --environment-variables를 포함해야 유지됨.
> 빠뜨리면 null로 초기화됨.

## 문제 12. Lambda 한글 인코딩 오류

**에러**
```
'ascii' codec can't encode characters in position 7-9:
ordinal not in range(128)
```

**원인**
Lambda에 payload 전송 시 한글이 포함된 경우
기본 `json.dumps()`가 ascii로 인코딩 시도.

**해결**
```python
# 수정 전
Payload=json.dumps(payload)

# 수정 후
Payload=json.dumps(payload, ensure_ascii=False).encode("utf-8")
```

## 문제 13. OpenAI API 키 채팅 노출로 자동 revoke

**원인**
채팅창에 API 키를 직접 붙여넣으면 OpenAI가 자동으로 감지해서 비활성화.

**해결**
새 키 발급 후 `.env` 파일에만 저장.
```bash
# AgentCore 환경변수 업데이트 시 .env에서 읽어서 주입
--environment-variables "{...,\"OPENAI_API_KEY\":\"$(grep OPENAI_API_KEY .env | cut -d= -f2)\",...}"
```

> ⚠️ API 키는 절대 터미널이나 채팅에 직접 입력하지 말 것.

## 문제 14. CloudWatch 로그 미수집

**원인**
agentcore-runtime-role에 CloudWatch 권한 없음.
컨테이너가 에러를 뱉어도 로그 그룹에 기록 안 됨.

**해결**
```bash
aws iam attach-role-policy \
  --role-name agentcore-runtime-role \
  --policy-arn arn:aws:iam::aws:policy/CloudWatchLogsFullAccess
```

## 디버깅 핵심 방법

CloudWatch 로그가 없을 때 컨테이너 내부 직접 진입해서 확인.

```bash
# ECR 이미지를 로컬에서 실행하고 내부 진입
docker run -it --entrypoint /bin/bash \
  -e 환경변수들... \
  070238434919.dkr.ecr.ap-northeast-2.amazonaws.com/analysis-agent:latest

# 컨테이너 내부에서 직접 테스트
python3 -c "
import chromadb
client = chromadb.PersistentClient(path='/app/lpi_vector_db')
col = client.get_collection('lpi_interactions')
print('성공:', col.count())
"
```

이 방법으로 chromadb 문제를 발견하고 해결할 수 있었음.

## 최종 Dockerfile

```dockerfile

WORKDIR /app

RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

RUN pip install --no-cache-dir -r requirements.txt

COPY app/ ./app/
COPY lpi_vector_db/ ./lpi_vector_db/
```
