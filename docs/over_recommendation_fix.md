# 과잉 추천 문제 수정 내역

## 문제 정의

### 증상
혈액검사 결과가 모두 정상 범위임에도 에이전트가 비타민D·철분·B12 등을 결핍으로 판정하고 영양제를 과잉 추천하는 현상 (False Positive).

### 재현 케이스
**골든셋 A-003** — 30대 남성, 혈액검진 전 항목 정상, 섭취 목적: "피로 개선"
- 기대 동작: `required_nutrients` 0개, `lifestyle_notes`에 피로 개선 생활습관 조언
- 실제 동작: 비타민D·B12·철분을 결핍으로 판정 → 3개 이상 영양제 추천

### 근본 원인
LLM의 학습 편향 — 사용자가 피로·면역 등 증상을 언급하면 **"뭔가 추천해야 한다"는 방향으로 추론**하는 경향이 있음. 혈액검사 수치가 정상 범위 하단에 있으면 이를 "낮은 정상"으로 해석해 결핍으로 판정함.

## 수정 내역

### Commit 1: `d98f01c`
**feat: Step1 프롬프트에 근거 등급 규칙 추가 — 과잉 추천 방지**

`SYSTEM_PROMPT_STEP1`에 **근거 등급(Evidence Grade) 체계** 도입.

각 영양소의 `reason` 필드에 세 가지 근거 등급 prefix 중 하나를 반드시 명시하도록 강제:

```
[혈액검사 근거]: 혈액검사 수치가 결핍 기준을 명확히 초과한 경우
[약물 상호작용]: 복용 중인 약물이 해당 영양소를 고갈·차단하는 경우
[섭취 목적]: 섭취 목적·증상과 연관성이 있으나 혈액검사 수치 근거 없음
```

**추천 개수 제한 규칙 추가:**
- `[혈액검사 근거]`, `[약물 상호작용]` 영양소는 개수 제한 없음
- `[섭취 목적]` 영양소는 사용자가 명시한 섭취 목적의 개수만큼만 허용
  - 섭취 목적 1개 → `[섭취 목적]` 영양소 최대 1개
  - 섭취 목적 3개 → `[섭취 목적]` 영양소 최대 3개
- 건강검진 데이터 없는 경우: `[섭취 목적]` 영양소 최대 3개로 제한

### Commit 2: `af4d74a`
**feat: 혈액검사 정상 시 영양제 대신 생활습관 조언으로 전환**

혈액검사 제공됐고 **결핍 항목이 전혀 없는 경우**에 대한 특별 규칙 추가:

```
건강검진 데이터가 제공됐고 결핍 항목이 전혀 없는 경우 (특별 규칙):
- [섭취 목적] 영양소를 required_nutrients에 포함하지 않는다
- 대신 summary.lifestyle_notes에 섭취 목적에 맞는 식이·운동·수면 조언을 구체적으로 작성한다
- summary.overall_assessment에는 반드시 아래 흐름으로 작성한다:
  1) 사용자의 섭취 목적을 먼저 언급한다
  2) 건강검진 결과 해당 부분에 이상이 없음을 알린다
  3) 영양제보다는 생활습관 개선이 더 효과적일 수 있음을 안내한다
  4) lifestyle_notes의 구체적 조언으로 자연스럽게 연결한다
```

**UX 개선 의도:** 사용자가 건강 목표를 말했을 때 무조건 영양제로 연결하지 않고, 검사 결과상 이상이 없으면 생활습관 개선을 먼저 안내하는 방향으로 전환.
